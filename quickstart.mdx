The fastest way to see Gecko work is the **recorded demo**: it runs the *same code
path* as live, but synthesizes responses from the API's schema instead of hitting the
network. No keys, no subscription, no spend. You can falsify the comprehension offline
before going live.

## Prerequisites

- Python 3.11+
- [`uv`](https://docs.astral.sh/uv/)

## Run the $0 recorded demo

```bash
git clone https://github.com/GeckoVision/surfcall
cd surfcall && uv sync
uv run pytest                       # 31+ passing
uv run python -m surfcall.demo      # E2E: goal → discover → correct call → data (recorded, $0)
```

The demo takes natural-language goals (e.g. *"get the latest live odds for a football
fixture"*), discovers the right endpoint, fills the required parameters, and returns a
schema-shaped sample — entirely offline.

<Note>
Recorded mode and the test suite need **no** API keys and **no** subscription. The
recorded response is synthesized from the spec's response schema, so it proves the
agent picked the right call and built a well-formed request — not that the upstream
returned that exact data.
</Note>

## Use it programmatically

`AgentApiClient` is the one object that makes an API agent-usable. Point it at an
OpenAPI document, search for a capability, and call it.

```python
from surfcall.client import AgentApiClient

# A spec path/URL or a parsed dict. With no session, a stub session is used,
# so auth-gated operations are still surfaced in recorded mode.
client = AgentApiClient("tests/fixtures/txodds_docs.yaml")

# intent → ranked endpoints
hits = client.search("get the latest live odds for a fixture")

# call the top hit in recorded mode ($0, offline)
result = client.call(hits[0]["name"], {"fixtureId": 123}, mode="recorded")
print(result["method"], result["request"], result["status"])
print(result["data"])   # schema-shaped sample
```

`search()` returns ranked entries (`name`, `summary`, `path`, `method`). `call()`
returns `{status, request, method, data, mode}`.

## Point it at a different public API

The engine is API-agnostic — adding an API is data (the spec) plus, at most, one
auth adapter. For a public, no-auth API, pass a `public_session()`:

```python
from surfcall.access import public_session
from surfcall.client import AgentApiClient

client = AgentApiClient("path/to/openapi.json", session=public_session())
print(len(client.list_tools()), "tools surfaced to the agent")
```

With a no-auth session, any operation that *requires* auth is automatically hidden
from the agent — it can't satisfy it, so it should never try to call it.

## Going live

Recorded mode needs nothing. For live data you pass a real `Session`; the call uses
the **same code path**, differing only at the transport edge:

```python
from surfcall.client import AgentApiClient

client = AgentApiClient(spec, base_url="https://...", session=my_session)
client.call(tool_name, args, mode="live")   # same path as recorded
```

See [Access & auth](/access-and-auth) for how a session is established and how the
two-token handshake works.
